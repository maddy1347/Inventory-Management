Method

Three steps, one pass

The same scoring pipeline runs whether you use this page or the command-line agent in the repo — down to the exact same Python file.

01 — Upload

Drop in your log

A CSV or Excel export with Date, Requests, Revenue, and either Fill Rate or Impressions. Supply Tag, Demand Tag, and App Bundle columns are auto-detected if present.

02 — Configure

Set your gates

Turn on a server-cost model, a revenue-floor override, or a global revenue-loss cap — or leave them all off and score on fill rate and revenue alone.

03 — Decide

Get the ladder position

Each tag gets a suggested action, a QPS change, a projected daily cap, and — for tags being throttled — a bundle-level blocklist within it.
